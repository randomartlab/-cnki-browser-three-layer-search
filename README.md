# cnki-browser-three-layer-search
替代垃圾CNKIAI的增强版skill，调用本地谷歌浏览器进行cnki检索，学术发现式搜索策略。

特色：

复刻学术训练基础能力——关键词发散式深层搜索；

① 在对话中指定一个或一组关注领域后，LLM生成一组概念簇，作为1层检索词；

② 调用本地有头谷歌浏览器进入CNKI（自行解决论文账号和IP权限），执行1层搜索；

③ 对单个关键词的搜索结果进行扫描（重点内容可通过指令深入到正文），分别与概念簇内或其他概念簇关键词进行匹配；

④ 形成认知，获得新关键词，进行2层单独关键词搜索（用于增强命中）

⑤ 继续进行匹配和验证

⑥ 每轮对话沉淀md文档，写入Nowledge Mem MCP。

## CNKI中文学术检索：三层渐进式关键词发现与逐篇关联判定

### 前置条件：MCP配置

在CodePilot自定义MCP中设置cnki-browser，指向本地有头Chrome：
- `--executablePath=你的Chrome完整路径`（Windows示例：`C:\Program Files\Google\Chrome\Application\chrome.exe`）
- `--headless=false`（必须有头模式）
- `--userDataDir` 使用独立目录，不要与CodePilot默认chrome-devtools共享
- MCP名称建议为 `cnki-browser`，不要覆盖默认chrome-devtools

### 前置条件：运行时规则

- 登录由用户在浏览器中自行完成，不得记录账号凭据
- 页面快照可能包含预加载或离屏的验证码组件；以用户可见界面和搜索框实际可操作状态判断是否存在验证阻塞，不凭可访问性树中的验证码文本判定
- 下载PDF按任务需要执行；批量或付费下载前先征求用户确认

### 检索流程

#### 第一步：构建概念树

为每个上位概念（研究对象的核心维度）建立子关键词树。子词应包括具体技术术语、相近概念、不同表述和常见翻译。每个维度独立建树。

示例（数字媒介维度）：虚拟现实、VR、增强现实、AR、扩展现实、XR、交互技术、数字场景、全息影像、数字孪生、人工智能、社交媒体、短视频等。

#### 第二步：三层渐进式关键词发现

**第一层**：使用单一的初步核心关键词宽检。禁止将多个关键词用空格拼入同一搜索框，这会过度限制检索广度。记录宽检结果的总量、文献类型和学科分布。

**第二层**：从第一层结果中进入重点关注论文，读取摘要、关键词、目录和参考文献，从中发现新的关键词和术语。每个新关键词分别独立检索，不与其他关键词组合约束。

**第三层**：进入第二层确定的关键论文，发现第三层术语、理论来源和引文，继续独立检索和回溯。完整执行3层。

#### 第三步：子关键词独立宽检与关联判定

每个子关键词独立执行宽检：
- 从结果题名、摘要、关键词和学科分类中识别与研究对象实质相关的论文
- 不以论文是否出现上位词为唯一关联判定标准
- 对每篇进入论文，区分处理深度：metadata only（仅阅元数据）、abstract only（仅阅摘要）、full text read（完整阅读）

#### 第四步：合并去重与关联编码

将各子词命中的相关文献取并集并去重。按关联类型编码归类，例如：技术应用、理论框架、实证研究、观众体验、产业背景等。

#### 第五步：交叉验证（仅用于查漏）

只有在子词宽检并集建立后，才使用"对象词×子词"交叉搜索做查漏验证：
- 直接交叉命中量只能说明字面词组在主题字段中的共同出现量，不代表概念层面的研究覆盖率
- 严禁使用单一上位词交叉命中量估算整个概念簇的关联规模
- 严禁根据上位词交叉命中的低比例得出"相关研究稀少"或"交叉空白"等结论

### 学科口径控制

检索与综述必须保持目标学科的口径。其他学科（如传播学、营销学、旅游管理）的文献只用于补充机制解释或产业背景，不得主导学术史叙述。

### 学术史梳理规范

在综述中串联学术史时：
1. 明确代表性学者提出了什么关键观点、得到什么研究发现
2. 呈现这些观点之间如何承接、补充、分化或推进
3. 呈现研究问题的学术演进路径
4. 在演进链条中准确定位现有研究的解释边界与本课题的新进展
5. 不按主题分类罗列文献，不只写"某某研究了什么"
